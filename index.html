How it’s being built (The SoT Method)

BUS Core is coded using an **AI as junior dev team** approach, enforced by discipline:

  • A strict **Source-of-Truth (SoT)** text file defines the architecture and prevents drift.
  • Every change must pass a **smoke test** before it’s committed, keeping the core code stable and debt-free.
  • The system runs a modular **Microkernel + SQLite** core for items, vendors, and runs.
